A continuaci'on se detallan los requerimientos del sistema a construir. Se ha identificado cada requerimiento con un c'odigo 'unico que permite referenciar a dichos requerimientos desde el resto del documento.

\subsubsection{Requerimientos funcionales}


% requerimientos esenciales
\subsubsubsection{Esenciales}

\begin{itemize}

\item { \negrita{El sistema permite abrir el casino} \req{req:abrir_casino}

El casino podr'a ser abierto cuando se lo desee por un administrador del sistema. 
}


\item \negrita{El sistema permite cerrar el casino cuando no hay jugadores en el mismo} \req{req:cerrar_casino}

 El casino podr'a ser cerrado cuando se lo desee por un administrador del sistema, con la condici'on de que no existan jugadores dentro al momento de hacerlo.

\item \negrita{El sistema cuenta con juegos de tipo Tragamonedas } \req{req:existe_juego_tragamoneda}

 Un casino sin juegos no es un casino. Para poder conseguir que los jugadores juegen hay, entre otros, juegos de tipo Tragamonedas.

\item \negrita{El sistema cuenta con juegos de tipo Craps} \req{req:existe_juego_craps}

 Existir'an juegos de tipo Craps.

\item \negrita{El sistema permite a los jugadores ingresar al casino} \req{req:ingreso_al_casino}

 Los jugadores deben poder ingresar al casino utilizando para ello los datos de su cuenta.

\item \negrita{El sistema permite a los jugadores crear una nueva mesa de juego} \req{req:creacion_de_mesa}

 Los jugadores que no est'en en una mesa de juego deben poder crear una nueva mesa de juego del tipo que elijan.

\item \negrita{El sistema permite a los jugadores ingresar a una mesa de juego existente} \req{req:ingreso_a_mesa}

 Los jugadores que no est'en en una mesa de juego deben poder ingresar a una mesa existente cuando lo deseen.

\item \negrita{El sistema permite a los jugadores abandonar una mesa de juego}\req{req:salir_de_mesa}

 Los jugadores que est'en en una mesa de juego deben poder salir de la misma.

\item \negrita{El sistema se encargar'a de cerrar las mesas que no tengan jugadores}\req{req:auto_cerrar_mesas}

 Cuando una mesa de juego haya sido abandonada por el 'ultimo jugador que la utilizaba, la misma ser'a cerrada de forma autom'atica por el sistema.

\item \negrita{El sistema permite a los jugadores apostar a los juegos si poseen suficiente cr'edito} \req{req:apuesta_a_juegos}

 Los jugadores que se encuentren en una mesa de juego y posean el saldo necesario deben poder apostar.

\item \negrita{El sistema permite siempre a los jugadores vip apostar a los juegos} \req{req:apuesta_vip_a_juegos}

 Los jugadores vip que se encuentren en una mesa de juego deben poder apostar siempre, incluso si su saldo es insuficiente para realizar una apuesta (los jugadores vip pueden tener saldo negativo).

\end{itemize}


% requerimientos importantes
\subsubsubsection{Importantes}

\begin{itemize}

%% \req{}
%%  Poder ajustar las variables de probabilidad y ganancias de cada juego al igual que las probabilidades de ocurrencia de jugadas felices y todos ponen. De 'esta forma se puede asegurar la rentabilidad del casino.

\item \negrita{El sistema permite entrar y salir del ``modo dirigido''} \req{req:existe_modo_dirigido}

 Un administrador de sistema puede activar y desactivar el ``modo dirigido'', lo que le permite controlar el azar y asegurar las ganancias del casino.

\item \negrita{El sistema permite controlar los resultados de los juegos} \req{req:controlar_resultados}

 Un administrador del sistema en ``modo dirigido'' puede controlar los resultados de los juegos.

\item \negrita{El sistema permite controlar el tipo de jugada de los juegos} \req{req:controlar_tipo_jugada}

 Un administrador del sistema en ``modo dirigido'' puede controlar el tipo de jugada (``normal'', ``feliz'' 'o ``todos ponen'') de los juegos.

\item \negrita{El sistema permite la configuraci'on de las probabilidades de los resultados de los juegos}\req{req:conf_probabilidades_resultados}

 Un administrador de sistema puede configurar la probabilidad de ocurrencia de los distintos resultados de los juegos.

\item \negrita{El sistema permite la configuraci'on de las probabilidades de ocurrencia de las jugadas ``feliz'' y ``todos ponen''} \req{req:conf_probabilidades_jugadas_especiales}

 Un administrador de sistema puede configurar la probabilidad de ocurrencia de las jugadas especiales.

\item \negrita{El sistema permite la configuraci'on de la probabilidad de ganar el ``premio gordo progresivo'' de las m'aquinas tragamonedas} \req{req:conf_probabilidades_premio_progresivo}

 Un administrador de sistema puede configurar la probabilidad de ganar el ``premio gordo progresivo'' de un jugador que satisface los requisitos necesarios para intentar ganarlo.

\item \negrita{El sistema permite la configuraci'on del valor m'inimo del pozo progresivo de las Tragamonedas}\req{req:ajuste_de_pozo_progresivo}

 Un administrador de sistema puede ajustar el valor m'inimo posible para el pozo com'un progresivo de los juegos de m'aquinas tragamonedas.

\item \negrita{El sistema permite generar reportes con informaci'on del estado del casino y sus jugadores} \req{req:generar_reportes}

 Un administrador de sistema puede pedir reportes del estado del casino y sus jugadores para mantener control de la rentabilidad y ajustar correctamente los juegos y pozos.

\item \negrita{El sistema permite acceder al casino a m'as de un jugador a la vez desde la misma terminal}\req{req:acceso_multiple}

 Dos o m'as jugadores deben poder acceder al casino desde una misma terminal.

\item \negrita{El sistema permite ser extendido mediante la incorporaci'on de nuevos juegos} \req{req:agregar_juegos}

 Cuando los SOCIOS lo deseen, podr'an agregar nuevos juegos al sistema, ampliando las opciones para sus clientes.

\end{itemize}

% requerimientos deseables
\subsubsubsection{Deseables}

\begin{itemize}

\item \negrita{El sistema permite la comunicaci'on v'ia chat entre jugadores de una misma mesa} \req{req:chat}

 Los jugadores podr'an expresar entre s'i su felicidad por estar jugando.

\item \negrita{El sistema permite a los invitados ingresar y salir del casino} \req{req:inv_ingreso_egreso_al_casino}

 Los invitados deben poder entrar y salir del casino sin necesidad de poseer una cuenta de usuario.

\item \negrita{El sistema permite a los invitados mirar una mesa de juego existente} \req{req:inv_mirar_mesa}

 Los invitados que no est'en mirando una mesa de juego deben poder ingresar a una mesa existente cuando lo deseen.

\item \negrita{El sistema permite a los invitados abandonar una mesa de juego}\req{req:inv_salir_de_mesa}

 Los invitados que est'en mirando una mesa de juego deben poder abandonar la misma.

\end{itemize}



\subsubsection{Requerimientos no funcionales}

\begin{itemize}

\item \negrita{El sistema es seguro respecto a las transacciones de dinero} \req{req:sistema_seguro}

 Se necesita que las transacciones sean seguras para brindar mayor confianza a los jugadores y as'i fomentar el uso del casino.

\item \negrita{El sistema permite suficiente cantidad de jugadores} \req{req:suficientes_jugadores}

 El sistema deber'a poder adaptarse f'acilmente para permitir que el n'umero de jugadores sea el requerido para satisfacer las necesidades de rentabilidad.

\item \negrita{Los juegos que el sistema ofrece son similares a los de la realidad} \req{req:realGames}

 Es necesario para que los jugadores se sientan c'omodos y les sea sencillo a los jugadores nuevos adaptarse al sistema inform'atico.

\end{itemize}
